# 30 比较本书与 HtDP

> 原文：[`dcic-world.org/2025-08-27/htdp-vs-dcic.html`](https://dcic-world.org/2025-08-27/htdp-vs-dcic.html)

这本书（DCIC）经常与[如何设计程序](https://www.htdp.org/)（HtDP）进行比较，它从中汲取了巨大的灵感。在这里，我们简要描述了两本书的比较。

从高层次来看，它们非常相似：

+   两本书都是围绕数据结构的中心构建的。它们都希望提供设计程序的方法。它们从函数式编程开始，过渡到（并且非常认真地对待）有状态的命令式编程。

+   两本书都是围绕教育理念精心设计的语言构建的。这些语言提供了编写示例和测试的特殊支持；为初学者设计的错误报告；内置图像和反应性。这些语言避免了奇怪的问题（在 Python 中并不是这样：参见 Pyret vs. Python 或，如果你想读更多，[这篇论文](https://cs.brown.edu/~sk/Publications/Papers/Published/pmmwplck-python-full-monty/)))。

以此类推。然而，将这些称为“相似之处”是不公平的。DCIC 是从 HtDP 那里复制了这些想法；在某些情况下，HtDP 甚至开创了这些想法。

现在来看看差异。请注意，这些现在是差异。DCIC 的一些想法将进入 HtDP，随着时间的推移，可能会有更多相互融合。

+   最明显的是，DCIC 使用 Pyret。HtDP 有很多好想法，但由于它使用 Racket（一些人对 Racket 的语法不喜欢，尤其是教育工作者），所有这些想法都被忽视了。我们构建 Pyret 是为了体现我们从 Racket 学生语言和其他好的想法中学到的知识，但用熟悉的语法包装它们。但正如你所看到的，这两种语言实际上并没有那么远：Pyret for Racketeers and Schemers。

+   下一个最明显的事情是，DCIC 还包含了 Python。HtDP 有一个（非正式出版的）后续作品[教授 Java 程序设计](https://felleisen.org/matthias/HtDC/htdc.pdf)。相比之下，我们希望将 Python 的过渡整合到 DCIC 本身中。从对比中可以学到很多东西！特别是，Pyret 及其环境是围绕教学状态的教学理念精心设计的。尽管状态无处不在且难以处理，但 Python 却没有。因此，在引入状态时，对比它们可以获得很多收获。

+   接下来，DCIC 有很多算法内容，而 HtDP 几乎没有。例如，DCIC 涵盖了 Big-O 分析[预测增长]。它甚至有一个关于摊销分析的章节[万圣节分析]。它还涉及一些图算法。这比 HtDP 涵盖的内容要高级得多。

这些是大部分的差异。它们从目录表中就可以明显看出（有些甚至一目了然）。然而，有一个非常深刻的差异，大多数读者可能不会注意到，我们将在下面讨论。

HtDP 是围绕一个美好的理念构建的：展示的数据结构在集合论术语中逐渐增加复杂性。因此，它从原子数据开始，然后是固定大小的数据（结构），然后是无界集合（列表）的原子数据，列表对，结构列表，等等。所有这些都有序地、系统地构建起来。

然而，这也带来了一些负面影响。你必须想象数据代表什么（这个数字是年龄，那个字符串是名字，那个列表是 GDP），但它们是理想化的。从某种意义上说，最真实的数据实际上是图像！在那之后（这些出现在早期），所有数据都是“虚拟化”和想象的。

我们的看法是，最有趣的数据是结构列表。（还记得吗？它们很复杂，位于进阶的某个阶段。）你可能会觉得这很惊讶；如果是这样，我们给你另一个名字：表格。表格无处不在。即使是公司也会处理和发布它们；即使是小学生也能识别和使用它们。它们可能是我们最重要的通用结构化数据形式。

更好的是，提供了大量真实世界的数据作为表格。你不必想象事情或编造虚假的 GDP，比如 1、2 和 3。你可以得到实际的 GDP 或人口或电影收入或体育排名或任何你感兴趣的东西。（理想情况下，经过净化和整理。）我们相信，几乎每个学生——甚至每个孩子——都是一个潜在的数据科学家（至少在他们方便的时候）。即使是说“我讨厌数学”的孩子，也常常乐意使用统计数据来为自己的最喜欢的演员或运动员或任何东西辩护。我们只需要找到激发他们的动机。

但是有一个大问题！HtDP 的一个关键特性是，对于每一种数据类型，它都提供了一个针对该数据类型的编程设计食谱。结构列表很复杂。它们的编程食谱也是如此。我们希望将它们放在开头！此外，忽视设计食谱是危险的。学生们在空白页面上挣扎，常常填满糟糕的代码，然后对它们产生依恋。设计食谱提供了结构、支架、可审查性以及更多。它在认知上基于模式。

所以在过去的几年里，我们一直在研究不同的程序设计方法，这些方法通过不同的手段达到相同的目标。我们最近的教育研究工作在很大程度上是在建立新的基础。这是一项正在进行中的工作。DCIC 就是这些努力的结晶。随着我们获得新的成果，我们将把它们编织进 DCIC（也许还有 HtDP）。请保持关注！
