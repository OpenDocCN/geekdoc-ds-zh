# 31 版本说明

> 原文：[`dcic-world.org/2025-08-27/Release_Notes.html`](https://dcic-world.org/2025-08-27/Release_Notes.html)

这本书的每个版本更新摘要（不包括错别字和其他小修复）。

### 31.1 版本 2025-08-27 "链接至此")

本版本有几处虽小但可能重要的更改：

+   在处理表格中用作示例的表格事件表现在增加了一个带有邮政编码的列。邮政编码提出了数据看似数字但实际上应按类别处理的想法。

    没有邮政编码的表格版本在书的早期版本中使用的 URL 和 Google Sheet ID 上保持不变（因此任何之前的代码将继续工作）。2025-08-27 版本的代码片段已更新，包括邮政编码列。它们还使用对新版本的 Google Sheet 和包含该列的 CSV 文件的引用。

+   前言包括在 VSCode 中使用 Pyret 的信息（作为 CPO 的替代方案）。这两个工具之间的唯一区别在于如何从外部源导入表格。CPO 从 Google Sheets 导入表格，而 VSCode 从 CSV 文件（通过 URL 或文件路径）导入表格。

    本书使用 CPO 作为默认演示。当代码示例只能在 VSCode 中运行（特别是如果它们导入 CSV 文件）时，代码周围将环绕一个边框和 VSCode 标志。

+   在从 Pyret 到 Python 的末尾添加了关于 Pyret 中 `for each` 迭代器的章节。这种放置支持那些在 Pyret 中学习如何处理列表后希望转向 Python 的课程，而不会打断那些在覆盖树之后转向 Python 的课程的流程（最初设想的流程）。

    在树之后转向 Python 的课程可能希望跳过 Pyret 的 `for each` 材料。

+   书的从 Pyret 到 Python 部分末尾新增了一章关于 Python 中的文件输入和输出。

    由于这一新增内容，从编程状态（在 Pyret 和 Python 中）部分开始的后续章节编号都增加了 1。

+   编程状态（在 Pyret 和 Python 中）的两个主要部分已被重命名为可变结构和可变变量，以使它们的意图更清晰。为了保持该部分和子标题与“突变”术语的一致性，我们还将其中的几个标题重命名为使用“突变”一词，而不是之前使用的更口语化的“更新”。

    更改章节名称也会改变到达这些子章节的 URL。现有的链接到这些章节将继续工作，因为书籍中的 URL 包含发布标签。然而，任何计划更新链接以引用 2025-08-27 发布版本的读者应重新检查使用状态编程（在 Pyret 和 Python 中）章节中的 URL。

### 31.2 版本 2025-02-09 "链接至此")

本版本引入了一个新的上下文，`dcic2024`，其中包含所有之前在共享驱动器文件中的表格处理函数。如何选择上下文的说明在介绍表格的章节中。

错误纠正：

+   在程序目录中，更正了当程序尝试重新定义/为现有标识符赋予新值时 Pyret 报告的内容的描述。之前的文本提到了阴影，而不是冲突的值。

+   在处理缺失条目中，更正了关于`num-sanitizer`的错误描述，该描述之前声称它将缺失值默认为 0。增加了一个讨论，说明为什么这样的默认值可能是一个坏主意。

此外，在回顾：列表操作摘要的结尾提出的计算两个列表交集的挑战问题在介绍下一节中的`lam`之后得到了细化，并重新审视。

### 31.3 版本 2024-09-03 "链接至此")

### 31.4 版本 2023-02-21 "链接至此")

本版本有一系列重大更改：

+   本书已被拆分为一系列手册，以使其结构更清晰和组织。有关详细信息，请参阅材料组织。

+   早期版本中的几个大章节已被拆分为较小的章节，并分散在手册中。

+   编程入门材料已大幅修订和扩展。

+   材料顺序已更改。关于算法分析的材料已被移至 Python 材料之后。

+   从 Pyret 到 Python 的过渡得到了显著改进。

+   现在有一个关于使用 Pandas 的章节，它基于表格的相应 Pyret 示例。

+   Python 字典已移动到 From Pyret to Python。尽管我们没有涵盖 Pyret 的字典，但将字典材料放在状态之前对于学生学习 Python 来说是一个更自然的流程。然后我们在 Hashes, Sets, and Key-Values 中使用状态来开发字典的实现。

+   现在有一个全新的统一方法来 Programming With State，该方法将 Pyret 和 Python 版本并排展示。在两种不同的语言中看到这些材料有助于关注相似之处和差异，并可以改善语言之间的迁移。

+   依赖于算法分析的材料现在已与不依赖算法分析的材料分开。此外，与本书的主题保持一致，重点再次放在数据结构上。结果是，在 Data Structures with Analysis 中，对大量的先前材料进行了重构，使其更加清晰。

+   一些材料已被进一步重构为 Advanced Topics。此外，还有更多新的高级内容！
