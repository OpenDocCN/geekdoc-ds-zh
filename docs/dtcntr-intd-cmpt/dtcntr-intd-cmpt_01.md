### 1.1 这本书是关于什么的🔗 "链接至此")

这本书是计算机科学的入门。它将教会你编程，并以实用价值和重要性进行编程。然而，它还将超越编程，进入计算机科学，这是一个丰富、深刻、迷人且美丽的知识学科。你将学会许多可以直接应用的有用事物，但我们也会向你展示一些隐藏在背后和超越的东西。

最重要的是，我们希望提供一些思考问题解决方法的方式，这些方法使用计算。其中一些是技术方法，例如从数据和示例开始构建问题的解决方案。其他是科学方法，例如确保程序可靠并执行它们所声称的功能的方法。最后，还有一些是社会性的，思考程序对人们的影响。

### 1.2 驱动这本书的价值观🔗 "链接至此")

我们的观点受到我们作为软件开发者、研究人员和教育工作者数十年的经验指导。这在我们心中培养了以下信念：

+   软件不仅是为了运行而编写的。它还必须是为了被他人阅读和维护而编写的。通常，“其他人”是你，六个月后的你，你已经忘记了你做了什么以及为什么这么做。

+   程序员负责确保他们的软件达到预期的目标并且是可靠的。这在计算机科学的多个学科中都有所体现，例如测试和验证。

+   程序应该易于预测。在程序运行之前，我们需要尽可能多地了解它的行为。这种行为不仅包括技术特征，如运行时间、空间、电力等，还包括社会影响、利益和危害。程序员在思考这些方面一直表现得非常糟糕。

### 1.3 我们对数据的角度🔗 "链接至此")

这些担忧与我们对计算机科学作为学科如何发展的信念相交。我们生活在一个数据泛滥的世界是一个不言而喻的事实，但这有什么后果呢？

在计算层面，数据产生了深远的影响。传统上，唯一使程序变得更好的方法就是直接改进程序，这通常意味着使程序更加复杂，影响我们上面讨论的价值观。但是，对于某些程序类，还有另一种方法：只需给相同的程序提供更多或更好的数据，程序就可以改进。这些数据驱动程序是我们周围许多创新的核心。

除了这种技术效果之外，数据还可以产生深远的教学影响。大多数入门编程课程都受到没有实际意义、兴趣或后果的人工数据的困扰（并且通常还有与之相伴的人工问题）。有了真实数据，学习者可以个性化他们的教育，专注于他们认为有意义、丰富或仅仅是好玩的问题——提出和回答他们认为有价值的问题。确实，从这个角度来看，程序是询问数据的工具：也就是说，程序是回答问题的工具。反过来，对真实数据和真实问题的强调使我们能够讨论计算的社会影响。

这些现象催生了全新的研究领域，通常被称为数据科学。然而，典型的数据科学课程也存在许多局限性。它们很少关注我们关于学习编程的困难的认识。它们对软件可靠性重视不足。并且它们未能认识到它们的数据在结构上往往相当有限。这些局限性，即数据科学通常结束的地方，是计算机科学开始的地方。特别是，数据结构是思考并实现上述一些价值观——性能、可靠性和可预测性——的出发点——使用计算机科学的许多工具。

### 1.4 这本书的独特之处🔗 "链接至此")

首先，我们提出了一种新的构建计算机课程结构的方法，我们称之为数据中心性。关于这一点，请阅读我们的文章[我们的文章](https://cs.brown.edu/~sk/Publications/Papers/Published/kf-data-centric/)。我们认为以数据为中心的课程是这样的

> 数据中心性 = 数据科学 + 数据结构

按此顺序：我们首先从数据科学的思想开始，然后转向计算机科学中的经典数据结构思想。这本书具体而详细地阐述了这一愿景。

其次，计算机教育经常讨论概念机——<wbr>程序行为的抽象，旨在帮助学生理解程序是如何工作的——<wbr>但很少有课程真正使用它们。我们认真对待概念机，开发了一系列概念机并将它们贯穿于整个课程中。这符合我们的信念，即程序不仅仅是运行的对象，也是我们推理的对象。

第三，我们将关于负责任计算的内容编织到文本中。与其他专注于向学生展示伦理或技术一般陷阱的努力不同，我们旨在展示学生如何将他们现在正在将其转化为代码的结构和概念，如果不小心使用，可能会导致不良影响。遵循我们对测试和具体例子的关注，我们通过让学生思考具体数据层面的假设来引入几个主题。这些材料在整本书中都有明确的提及。

最后，这本书深受最近和正在进行的研究成果的启发。我们选择材料、展示顺序、编程方法等方面的选择都受到研究文献中我们所知的影响。在许多情况下，我们自己就是进行研究的那些人，因此课程和研究处于共生关系。你可以在我们的[论文](https://cs.brown.edu/~kfisler/Pubs/index.html)（有些是相互的，有些则不是）[页面](https://cs.brown.edu/~sk/Publications/Papers/Published/) [上](https://www.ccs.neu.edu/home/blerner/papers.html) [找到](https://jpolitz.github.io)。

### 1.5 这本书面向的对象🔗 "链接到此处")

这本书主要面向那些处于高等教育（大学或学院）计算机教育初期阶段的学生。然而，许多——<wbr>特别是早期——<wbr>部分也适合中等教育（例如，在美国，大约是 6 至 12 年级，或 12 至 18 岁）。实际上，我们认为中等教育和高等教育之间存在自然的连续性，并认为这本书可以作为两者之间有用的桥梁。

### 1.6 这本书的结构🔗 "链接到此处")

与其他一些教科书不同，这本书不遵循自上而下的叙述方式。相反，它更像是一场对话，带有回溯。我们通常会逐步构建程序，就像一对程序员一样。我们会包含错误，不是因为不知道更好的方法，而是因为这是你学习的最佳方式。包含错误使得你无法被动地阅读：你必须与材料互动，因为你永远无法确定你所阅读内容的真实性。

最后，你总是会得到正确的答案。然而，这种非线性的路径在短期内可能会更加令人沮丧（你可能会想，“就直接告诉我答案吧！”），这使得这本书成为一个糟糕的参考指南（你不能打开到任意一页并确信它说的是正确的）。然而，那种挫败感正是学习的感受。我们不知道有其他方法可以绕过它。

我们使用视觉格式化来突出一些这些点。因此，在几个地方你会遇到这个：

> 练习
> 
> > 这是一个练习。请务必尝试一下。

这是一个传统的教科书练习。这是你需要自己完成的事情。如果你将这本书作为课程的一部分使用，这很可能已经被分配为家庭作业。相比之下，你也会找到类似这样的练习题：

> 现在就做！
> 
> > 这里有一个活动！你看到了吗？

当你遇到这样的问题时，请停下来。在继续之前，阅读、思考并制定一个答案。你必须这样做，因为实际上这是一个练习，但答案已经在书中了——通常在紧随其后的文本中（即，你现在正在阅读的部分）——或者通过运行程序你可以自己确定。如果你只是继续阅读，你会在没有思考的情况下看到答案（或者根本看不到，如果指示是运行程序），这样你既不能（a）测试你的知识，也不能（b）提高你的直觉。换句话说，这些都是额外的、明确的鼓励主动学习的尝试。然而，最终，我们只能鼓励它；练习它取决于你。

程序设计和开发的具体策略在类似这样的框中突出显示：

> 策略：如何……
> 
> > 这里是如何做某事的总结。

最后，我们还要指出具有视觉上独特区域的社会责任计算内容，例如：

> 责任计算：你考虑过……
> 
> > 这里有一些使用材料时可能遇到的社会陷阱。

### 1.7 材料组织🔗 "链接到此处")

因为这本书涵盖了在美国大学本科教育中通常需要多个学期才能学到的内容，所以我们将其分为七个主要分册。后续的分册依赖于一些早期的分册，但早期的分册可以作为一个独立的书籍来处理，为那些不继续学习的学生的课程提供一个令人满意的结局。

1.  编程入门：为初学者提供的编程和基本数据分析入门。它通过组合图像和处理表格来介绍核心编程概念，然后在介绍列表和树之前。本节中的虚拟机器基于替换。

    依赖项：无！

1.  从 Pyret 到 Python：学生学会将他们的知识从 Pyret 转移到 Python，突出显示两种语言及其传统编程风格（“范例”）之间的相似性和差异性。学生还介绍了 Pandas，作为一个现实世界的表格处理系统。

    依赖项：编程入门。

1.  使用状态编程：学生学会状态和别名的高级技巧。大部分内容都在 Python 和 Pyret 中都有涉及。这种对比让学生理解了多种语言如何处理相同的话题；实际上底层想法的相同之处；但也提供了一些关键差异，这些差异提供了洞察。概念机器通过将命名环境（这里称为目录）与结构化数据值的堆栈分离来扩展以涵盖状态和别名。

    依赖项：编程入门是必需的。从 Pyret 到 Python 有助于跟随本小册子的 Python 部分，但学生可以在没有见过 Python 的情况下完成 Pyret 部分。

1.  算法分析：学生被介绍到分析算法的多种技术。

    依赖项：编程入门。没有对状态的需求。

1.  带分析的数据结构：学生通过算法分析的角度了解更高级的数据结构，这激发了他们的修订和变化。

    依赖项：编程入门和算法分析在所有地方都是必需的。使用状态编程对于某些材料是必要的。

1.  高级主题：学生在程序设计和算法编程中覆盖了一系列有趣的计算机科学主题。相对于其他材料，这些内容要么更微妙，要么更高级，要么在主流课程中不那么重要。

    依赖项：所有材料都依赖于编程入门。某些材料依赖于算法分析和/或使用状态编程。

1.  交互式程序：学生可以用相对较少的依赖项编写交互式程序！

    依赖项：编程入门。

这种分解成小册子的方式允许在提供多种不同类型的课程时具有灵活性，这些课程在复杂程度上差异很大。例如，我们已经通过重新组合这些材料提供了两个非常不同的课程，其他人也可以跟随：

+   一门入门课程可以使用编程入门、从 Pyret 到 Python 和状态编程来涵盖计算机科学以数据为中心的视角，并使学生掌握 Python 的基本技能。这对应于布朗大学的[CSCI 0111](https://cs.brown.edu/courses/csci0111/)课程。

+   一门更高级的课程可以从编程入门开始，然后进行算法分析（可能逐步进行），接着是带分析的数据结构，在返回状态编程之前，穿插高级主题的内容。这对应于布朗大学的[CSCI 0190](https://cs.brown.edu/courses/csci0190/)课程。

课程页面存档了所有先前的课程实例，包括所有作业和相关材料。读者欢迎在自己的课程中使用这些材料。

许多这些课程将会有之前使用状态编程（在 Python、Java、Scratch 或其他语言中）的学生入学。根据我们的经验，这些学生中大多数人都得到了要么非常不完整，要么完全误导的状态解释和隐喻（例如，“变量是一个盒子”）。因此，他们对状态的理解仅限于绝对的基础，尤其是在他们接触到别名等重要主题时。结果，许多这些学生发现通过我们的概念机器正确理解状态是如何真正工作的既新颖又深刻。因此，我们建议慢而仔细地学习这些材料。

我们当然邀请读者创建他们自己章节内的章节混合设计。我们很乐意听到其他人的设计。

### 1.8 我们的编程语言选择🔗 "链接到此处")

如果我们想致富，我们会把这本书完全用 Python 写成。截至本书写作时，Python 正处于其教学用途的黄金时期（就像之前的 Java、C++、C、Pascal 等一样）。确实，Python 有许多吸引人的特点，其中最引人注目的是它在职位列表旁边的存在。然而，我们反复被 Python 作为学习编程的入门点所挫败。

因此，这本书介绍了两种编程语言。它从一个名为[Pyret](https://www.pyret.org/)的语言开始，这是我们为了解决我们的需求和挫折而设计的。它被明确设计为与本书中的编程风格相匹配，因此两者可以和谐地共同成长。它借鉴了 Python，但也借鉴了许多其他优秀的编程语言。因此，初学者可以放心，他们正在得到照顾，而对于那些对语言动物园（从蛇到骆驼）有所了解的程序员来说，Pyret 应该既熟悉又舒适。

然后，认识到 Python 作为标准通信语言以及其广泛库的价值，本书的使用状态编程（在 Pyret 和 Python 中）部分明确涵盖了 Python。我们不是从 Python 的零开始，而是从早期材料中系统地、逐步地过渡到 Python。我们相信这将使你比只看到一种编程语言更好地学习通用编程。然而，我们也相信这将帮助你更好地理解 Python：就像你一旦走出国门，接触到其他语言、国家或文化，就会更好地欣赏自己的语言、国家或文化一样。

### 1.9 编程工具🔗 "链接到此处")

[CPO](https://code.pyret.org/)（code.pyret.org 的缩写）是我们为 Pyret 的默认编程环境。它完全在浏览器中运行，并使用 Google Drive 进行身份验证和文件存储。

Pyret 也支持 VSCode。要安装它，请访问[`code.visualstudio.com`](https://code.visualstudio.com)。安装完成后，点击左侧的扩展选项卡，搜索 Pyret。你应该安装“Pyret Interactive Editor for VSCode”。

CPO 和 VSCode 在加载电子表格数据方面的支持上有所不同。CPO 旨在与 Google Sheets 一起工作，而 VSCode 不支持 Google Sheets，而是期望学生从本地 CSV 文件加载数据。除非书中另有说明，所有代码都可以在任一工具中运行，编辑和与程序交互的界面是相同的。如果有差异，书中将展示两种版本。在这种情况下，只能在 VSCode 中运行的代码将带有 VSCode 图标，如下所示：

> ```py
> include csv
> ```

我们不推荐任何特定的 Python 环境。任何允许你使用 pytest 和加载外部数据文件的 Python 编辑器都应该工作得很好。

### 1.1 本书的内容🔗 "链接到此处")

这本书是计算机科学的入门指南。它将教会你编程，并以实用价值和重要性为出发点进行编程。然而，它还将超越编程，深入到计算机科学这一丰富、深邃、迷人且美丽的知识学科。你将学会许多可以直接应用的有用知识，但我们也会向你展示一些隐藏在背后和超越其上的内容。

最重要的是，我们希望提供一些使用计算解决问题的思维方式。其中一些是技术方法，例如从数据和示例出发构建解决问题的方案。其他是科学方法，例如确保程序可靠并执行其声明的功能。最后，还有一些是社会性的，思考程序对人们的影响。

### 1.2 本书驱动的价值观🔗 "链接到此处")

我们的视角受我们作为软件开发者、研究人员和教育工作者数十年的经验指导。这在我们心中培养了以下信念：

+   软件不仅是为了运行而编写，还必须是为了被他人阅读和维护而编写。通常，“其他人”是你，六个月后的你，已经忘记了他们做了什么以及为什么这么做。

+   程序员负责确保他们的软件达到预期的目标并具有可靠性。这在计算机科学内部的多个学科中都有体现，如测试和验证。

+   程序应该易于预测。在程序运行之前，我们需要尽可能多地了解它的行为。这种行为不仅包括技术特征，如运行时间、空间、电力等，还包括社会影响、利益和危害。程序员在思考这些方面一直名声不佳。

### 1.3 我们对数据的看法🔗 "链接至此")

这些担忧与我们对计算机科学作为一门学科如何发展的信念相交。我们生活在一个数据泛滥的世界是一个不言而喻的事实，但这又有什么后果呢？

在计算层面，数据产生了深远的影响。传统上，唯一使程序变得更好的方法就是直接改进程序，这通常意味着使它更加复杂，并影响我们讨论的上述价值。但是，对于某些程序类，还有另一种方法：简单地给同一个程序提供更多或更好的数据，程序就可以改进。这些数据驱动程序是我们周围许多创新的核心。

除了这种技术影响之外，数据还可以产生深远的教学影响。大多数入门级编程都受到没有实际意义、兴趣或后果（以及通常伴随的人工问题）的人工数据的困扰。有了真实数据，学习者可以个性化他们的教育，关注他们认为有意义、丰富或纯粹有趣的问题——<wbr>提出和回答他们认为有价值的问题。事实上，从这个角度来看，程序是询问数据：也就是说，程序是回答问题的工具。反过来，对真实数据和真实问题的强调使我们能够讨论计算的社会影响。

这些现象催生了全新的研究领域，通常被称为数据科学。然而，典型的数据科学课程也存在许多局限性。它们很少关注我们关于学习编程的困难的认识。它们对软件可靠性的重视不足。而且，它们未能认识到它们的数据在结构上通常非常有限。这些局限性，即数据科学通常结束的地方，正是计算机科学开始的地方。特别是，数据结构作为思考并实现上述一些价值——<wbr>性能、可靠性和可预测性——<wbr>使用计算机科学的众多工具的出发点。

### 1.4 这本书的独特之处🔗 "链接至此")

首先，我们提出了一种新的关于构建计算机课程的观点，我们称之为数据中心性。有关更多信息，请阅读[我们的文章](https://cs.brown.edu/~sk/Publications/Papers/Published/kf-data-centric/)。我们认为以数据为中心的课程是

> 数据中心性 = 数据科学 + 数据结构

按照这个顺序：我们首先从数据科学的思想开始，然后转向计算机科学中的数据结构等经典思想。本书具体而详细地阐述了这一愿景。

其次，计算机教育经常讨论概念机器——<wbr>程序行为的抽象，旨在帮助学生理解程序是如何工作的——<wbr>但很少有课程真正使用它们。我们认真对待概念机器，开发了一系列概念机器并将它们贯穿于课程中。这符合我们的信念，即程序不仅是运行的对象，也是我们推理的对象。

第三，我们将关于负责任计算的内容编织到文本中。与其他专注于向学生展示伦理或技术一般陷阱的努力不同，我们的目标是展示学生如何将他们现在正在将它们转化为代码的结构和概念，如果不小心使用，可能会导致不良影响。遵循我们对测试和具体例子的关注，我们通过让学生思考具体数据层面的假设来引入几个主题。这些材料在整本书中都有明确的说明。

最后，本书深受最近和正在进行的研究成果的启发。我们选择材料、呈现顺序、编程方法等方面的选择都受到研究文献中我们所知的影响。在许多情况下，我们自己就是进行研究的那些人，因此课程和研究处于共生关系。您可以在我们的[论文](https://cs.brown.edu/~kfisler/Pubs/index.html)（有些是相互的，有些则不是）[页面](https://cs.brown.edu/~sk/Publications/Papers/Published/) [上](https://www.ccs.neu.edu/home/blerner/papers.html) [找到](https://jpolitz.github.io)。

### 1.5 本书面向对象🔗 "链接到此处")

本书主要面向那些处于高等教育（大学或学院）计算机教育早期阶段的学生。然而，它的许多——<wbr>特别是早期——<wbr>部分也适合中等教育（例如，在美国，大约是 6 至 12 年级，或 12 至 18 岁）。实际上，我们认为中等教育和高等教育之间存在自然的连续性，并认为这本书可以作为两者之间有用的桥梁。

### 1.6 本书结构🔗 "链接到此处")

与其他一些教科书不同，这本书不遵循自上而下的叙述。相反，它更像是一场对话的流程，带有回溯。我们经常会逐步构建程序，就像一对程序员一样。我们会包括错误，不是因为不知道更好的方法，而是因为这是您学习的最佳方式。包括错误使得您无法被动地阅读：您必须与材料互动，因为您永远无法确定您所阅读的内容的真实性。

最后，你总是会得到正确答案。然而，这种非线性的路径在短期内可能会更加令人沮丧（你可能会想，“直接告诉我答案吧！”），并且使这本书成为一个糟糕的参考指南（你不能打开到任意一页并确信它说的是正确的）。然而，那种挫败感正是学习的感受。我们不知道有其他方法可以绕过它。

我们使用视觉格式来突出显示这些点中的一些。因此，在几个地方你会遇到这种情况：

> 练习
> 
> > 这是一个练习。请尝试一下。

这是一个传统的教科书练习。你需要自己完成。如果你将这本书作为课程的一部分使用，这很可能已经被分配为家庭作业。相比之下，你还会找到类似这样的练习题：

> 现在就做！
> 
> > 这里有一个活动！你看到了吗？

当你遇到这些内容之一时，请停下来。在继续之前，先阅读、思考并形成答案。你必须这样做，因为实际上这是一个练习，但答案已经在书中了——通常在紧随其后的文本中（即，你现在正在阅读的部分）——或者是你通过运行程序可以自己确定的内容。如果你只是继续阅读，你会在没有思考的情况下看到答案（或者根本看不到，如果指示是运行程序），这样你既不能（a）测试你的知识，也不能（b）提高你的直觉。换句话说，这些都是鼓励主动学习的额外、明确尝试。然而，最终，我们只能鼓励它；是否练习它取决于你。

程序设计和开发的具体策略在类似这样的框中突出显示：

> 策略：如何……
> 
> > 这里是做某事的总结。

最后，我们还会用类似这样的视觉区分区域来指出负责任计算的内容：

> 负责任计算：你考虑过……
> 
> > 这里有一些使用材料时可能遇到的社会陷阱。

### 1.7 材料组织🔗 "链接到此处")

因为这本书涵盖了在美国被视为相当于多个学期的高等教育材料，所以我们将其分为七个主要手册。后面的手册依赖于一些早期的手册，但早期手册可以被视为一个独立的书籍，为不继续的学生或课程提供了一个令人满意的结局。

1.  编程入门：为初学者提供的编程和基本数据分析入门。它通过组合图像和处理表格来介绍核心编程概念，然后在介绍列表和树之前。本节中的虚拟机器基于替换。

    依赖项：无！

1.  从 Pyret 到 Python：学生学会将他们的知识从 Pyret 转移到 Python，突出显示两种语言及其传统编程风格（“范式”）之间的相似性和差异性。学生还介绍了 Pandas，作为一个现实世界的表格处理系统。

    依赖项：编程入门。

1.  状态编程：学生学习状态和别名的高级技巧。大部分内容都在 Python 和 Pyret 中都有涉及。这种对比让学生理解了多种语言如何处理相同的话题；实际上底层思想是如何相同的；同时也提供了一些关键差异，这些差异提供了洞察。概念机器通过将命名环境（这里称为目录）与结构化数据值的堆栈分离来扩展以覆盖状态和别名。

    依赖项：编程入门是必需的。从 Pyret 到 Python 有助于理解这本小册子的 Python 部分，但学生可以在没有见过 Python 的情况下完成 Pyret 部分。

1.  算法分析：学生被介绍到分析算法的多种技术。

    依赖项：编程入门。没有对状态的依赖。

1.  带分析的数结构：学生通过算法分析的角度接触更高级的数据结构，这激发了他们对这些结构的复习和变体。

    依赖项：编程入门和算法分析在所有地方都是必需的。状态编程对于某些材料是必要的。

1.  高级主题：学生在程序设计和算法编程中覆盖了一系列有趣的计算机科学主题。与其它材料相比，这部分内容要么更加微妙，要么更加高级，或者对于主流课程来说不那么重要。

    依赖项：所有材料都依赖于编程入门。一些材料依赖于算法分析和/或状态编程。

1.  交互式程序：学生可以用相对较少的依赖项编写交互式程序！

    依赖项：编程入门。

这种分解成小册子的方式允许在提供不同类型、不同复杂程度的课程时具有灵活性。例如，我们通过重新混合这些材料已经提供了两个非常不同的课程，其他人也可以效仿：

+   一门入门课程可以使用编程入门、从 Pyret 到 Python 和状态编程来涵盖计算机科学以数据为中心的视角，并让学生掌握 Python 的基本技能。这对应于布朗大学的[CSCI 0111](https://cs.brown.edu/courses/csci0111/)课程。

+   一门更高级的课程可以从编程入门开始，然后进行算法分析（可能逐步进行），接着是带分析的数据结构，在返回到状态编程之前，穿插高级主题的内容。这对应于布朗大学的[CSCI 0190](https://cs.brown.edu/courses/csci0190/)课程。

课程页面存档了所有之前的课程实例，包括所有作业和相关材料。读者欢迎在自己的课程中使用这些材料。

许多这些课程将会有之前使用状态编程（在 Python、Java、Scratch 或其他语言中）的学生入学。根据我们的经验，这些学生中大多数人都得到了要么非常不完整，要么完全误导的状态解释和隐喻（例如，“变量是一个盒子”）。因此，他们对状态的理解仅限于绝对的基础，特别是在他们接触到诸如别名等重要主题时。结果，许多这些学生发现通过我们的概念机器正确理解状态是如何真正工作的既新颖又深刻。因此，我们建议慢而仔细地学习这些材料。

当然，我们邀请读者在各个章节内创建他们自己的混合版本。我们很乐意听到其他人关于设计的想法。

### 1.8 我们的编程语言选择🔗 "链接到此处")

如果我们想变得富有，我们会用 Python 完全编写这本书。截至本书写作时，Python 正处于其教学用途的黄金时期（就像之前的 Java、C++、C、更早的 Pascal 等）。确实，Python 有许多吸引人的特点，其中最不重要的是它在职位列表旁边的存在。然而，我们反复对 Python 作为学习编程的入门点感到沮丧。🔗 "链接到此处"

因此，本书包含两种编程语言。它从一个名为 [Pyret](https://www.pyret.org/) 的语言开始，这是我们为了解决我们的需求和挫折而设计的。它被明确设计为适合本书中的编程风格，因此两者可以和谐发展。它借鉴了 Python，但也借鉴了许多其他优秀的编程语言。初学者可以放心，因为他们知道他们正在得到照顾，而对于那些对语言动物园（从蛇到骆驼）有所了解的程序员来说，Pyret 应该是熟悉和舒适的。

然后，认识到 Python 既是标准通信语言，又因其丰富的库而具有价值，本书的使用状态编程（在 Pyret 和 Python 中）部分明确涵盖了 Python。我们不是从零开始学习 Python，而是从早期材料中系统地、逐步地过渡到 Python。我们相信这将使你比只看到一种编程语言更好地学习通用编程。然而，我们也相信这将帮助你更好地理解 Python：就像你一旦走出国门，接触到其他语言、国家或文化，就会更好地欣赏自己的语言、国家或文化一样。

### 1.9 编程工具🔗 "链接至此")

[CPO](https://code.pyret.org/)（code.pyret.org 的缩写）是我们为 Pyret 定义的默认编程环境。它完全在浏览器中运行，并使用 Google Drive 进行身份验证和文件存储。

Pyret 也支持 VSCode。要安装它，请访问 [`code.visualstudio.com`](https://code.visualstudio.com)。安装完成后，点击左侧的扩展选项卡，搜索 Pyret。你应该安装“Pyret Interactive Editor for VSCode”。

CPO 和 VSCode 在加载电子表格数据方面的支持不同。CPO 设计用于与 Google Sheets 一起工作，而 VSCode 不支持 Google Sheets，而是期望学生从本地 CSV 文件加载数据。除非本书另有说明，否则所有代码都可以在任一工具中运行，编辑和与程序交互的界面是相同的。如果有差异，本书将展示两种版本。在这种情况下，只能在 VSCode 中运行的代码将带有 VSCode 图标，如下所示：

> ```py
> include csv
> ```

我们不推荐任何特定的 Python 环境。任何允许你使用 pytest 和加载外部数据文件的 Python 编辑器都应该工作得很好。

### 1.3 我们对数据的角度🔗 "链接到此处")

这些关注点与我们对计算机科学作为学科发展的信念相交。我们生活在一个数据泛滥的世界，这是一个众所周知的事实，但这会产生什么后果呢？

这些现象催生了全新的研究领域，通常被称为数据科学。然而，典型的数据科学课程也存在许多局限性。它们很少关注我们关于学习编程难度的了解。它们对软件可靠性的强调不足。而且，它们未能认识到它们的数据在结构上通常非常有限。这些局限性，即数据科学通常结束的地方，是计算机科学开始的地方。特别是，数据结构是思考并实现上述一些价值——性能、可靠性和可预测性——的出发点——<wbr>使用计算机科学的许多工具。

### 1.4 这本书的独特之处🔗 "链接到此处")

首先，我们提出了一种新的构建计算课程结构的方法，我们称之为数据中心性。关于这一点，请阅读[我们的文章](https://cs.brown.edu/~sk/Publications/Papers/Published/kf-data-centric/)。我们认为，以数据为中心的课程应该

> 数据为中心 = 数据科学 + 数据结构

按此顺序：我们首先从数据科学的思想开始，然后转向数据结构以及计算机科学的经典思想。这本书具体而详细地阐述了这一愿景。

其次，计算机教育谈论了很多关于概念机器——<wbr>程序行为的抽象，旨在帮助学生理解程序是如何工作的——<wbr>但很少有课程真正使用它们。我们认真对待概念机器，开发了一系列概念机器，并将它们编织到课程中。这与我们的信念相符，即程序不仅仅是运行的对象，也是我们可以对其进行推理的对象。

第三，我们将关于负责任计算的内容编织到文本中。与其他专注于向学生展示伦理或技术普遍陷阱的努力不同，我们的目标是展示学生如何将他们现在正在将其转化为代码的结构和概念，如果不小心使用，可能会导致不良影响。与我们对测试和具体例子的关注保持一致，我们通过让学生思考具体数据层面的假设来引入几个主题。这些材料在整本书中都有明确的提及。

最后，这本书深受最近和正在进行的研究成果的启发。我们选择材料、呈现顺序、编程方法等都是基于我们从研究文献中了解到的。在许多情况下，我们自己就是进行研究的那些人，因此课程和研究生活在一种共生关系中。你可以在我们的[页面](https://cs.brown.edu/~kfisler/Pubs/index.html)（有些是彼此的，有些则不是）[我们的](https://cs.brown.edu/~sk/Publications/Papers/Published/) [各自](https://www.ccs.neu.edu/home/blerner/papers.html) [页面](https://jpolitz.github.io)上找到我们的论文。

### 1.5 这本书面向谁🔗 "链接到此处")

这本书主要面向那些处于高等教育（大学或学院）计算机教育初期阶段的学生。然而，许多——尤其是早期——部分也适合中等教育（例如，在美国，大约是 6 至 12 年级，或 12 至 18 岁）。实际上，我们认为中等教育和高等教育之间存在一种自然的连续性，并认为这本书可以作为两者之间有用的桥梁。

### 1.6 这本书的结构🔗 "链接到此处")

与一些其他教科书不同，这本书并不遵循自上而下的叙述方式。相反，它更像是一场对话，带有回顾。我们经常会逐步构建程序，就像一对程序员一样。我们会包含错误，并不是因为我们不知道更好的方法，而是因为这是你学习的最佳方式。包含错误使得你无法被动地阅读：你必须与材料互动，因为你永远无法确定你所阅读内容的真实性。

最后，你总会得到正确的答案。然而，这种非线性的路径在短期内可能会更加令人沮丧（你经常会想，“就告诉我答案吧！”），并且使得这本书成为一个糟糕的参考指南（你无法打开到任意一页并确信其内容是正确的）。然而，那种挫败感正是学习的感受。我们不知道有其他方法可以绕过它。

我们使用视觉格式来突出一些这些点。因此，在几个地方你会遇到这种情况：

> 练习
> 
> > 这是一个练习。真的试试看。

这是一个传统的教科书练习。这是你需要自己完成的事情。如果你将这本书作为课程的一部分使用，这很可能已经被分配为家庭作业。相比之下，你也会找到类似练习的问题，看起来像这样：

> 现在就做！
> 
> > 这里有一个活动！你看到了吗？

当你遇到这些中的一个时，停下来。在继续之前，阅读、思考并制定一个答案。你必须这样做，因为实际上这是一个练习，但答案已经在书中了——<wbr>通常在紧随其后的文本中（即，你现在正在阅读的部分）——<wbr>或者通过运行程序你可以自己确定。如果你只是继续阅读，你会在没有思考的情况下看到答案（或者根本看不到，如果指示是运行程序），因此你既不能（a）测试你的知识，也不能（b）提高你的直觉。换句话说，这些都是鼓励主动学习的额外、明确尝试。然而，最终，我们只能鼓励它；是否练习它取决于你。

在类似这样的框中突出显示程序设计和开发的具体策略：

> 策略：如何...
> 
> > 这里是如何做某事的总结。

最后，我们还通过类似这样的视觉上独特的区域来强调负责任计算的内容：

> 负责任计算：你考虑过...
> 
> > 这里列出了使用材料时可能出现的社交陷阱。

### 1.7 材料组织🔗 "链接至此")

由于本书涵盖了在美国大学本科教育中通常需要多个学期才能学到的内容，因此我们将其分为七个主要分册。后续分册依赖于一些早期分册，但早期分册可以作为一个独立的书籍来处理，为那些不继续学习的学员或课程提供一个令人满意的结局。

1.  编程入门：为初学者提供的编程和基础数据分析入门。通过组合图像和处理表格来介绍核心编程概念，然后在介绍列表和树之前。本节中的虚拟机基于替换原理。

    依赖项：无！

1.  从 Pyret 到 Python：学生学会将他们的知识从 Pyret 转移到 Python，突出显示两种语言及其传统编程风格（“范式”）之间的相似之处和差异。学生还介绍了 Pandas，作为一个现实世界的表格处理系统。

    依赖项：编程入门。

1.  使用状态编程：学生学习状态和别名的高级技巧。大部分内容都在 Python 和 Pyret 中都有涉及。这种对比让学生了解多种语言如何处理相同主题；实际上底层思想的相同之处；但也提供了一些关键差异，这些差异提供了洞察力。虚拟机通过将命名环境（在此称为目录）与结构化数据值的堆栈分离，扩展到涵盖状态和别名。

    依赖项：编程入门是必需的。从 Pyret 到 Python 有助于跟随本手册的 Python 部分，但学生可以在没有见过 Python 的情况下完成 Pyret 部分。

1.  算法分析：学生被介绍到分析算法的多种技术。

    依赖项：编程入门。没有对状态的需求。

1.  带有分析的算法数据结构：学生通过算法分析的角度接触更高级的数据结构，这激发了他们的复习和变体。

    依赖项：编程入门和算法分析在所有地方都是必需的。状态编程对于某些材料是必要的。

1.  高级主题：学生在程序设计和算法编程中覆盖了一系列有趣的计算机科学主题。与其他材料相比，这些内容要么更微妙，要么更高级，要么在主流课程中不那么重要。

    依赖项：所有材料都依赖于编程入门。一些材料依赖于算法分析和/或状态编程。

1.  交互式程序：学生可以用相对较少的依赖项编写交互式程序！

    依赖项：编程入门。

这种分解为手册的方式，在提供不同复杂程度的多种不同类型的课程方面提供了灵活性。例如，我们通过重新组合这些材料已经提供了两个非常不同的课程，其他人也可以效仿：

+   一门入门课程可以使用编程入门、从 Pyret 到 Python 和状态编程来涵盖以数据为中心的计算机科学视角，并使学生掌握 Python 的基本技能。这对应于布朗大学的[CSCI 0111](https://cs.brown.edu/courses/csci0111/)课程。

+   一门更高级的课程可以从编程入门开始，然后进行算法分析（可能逐步进行），接着是带有分析的算法数据结构，在返回状态编程的同时，穿插高级主题的内容。这对应于布朗大学的[CSCI 0190](https://cs.brown.edu/courses/csci0190/)课程。

课程页面存档了所有课程的先前实例，包括所有作业和相关材料。读者欢迎在自己的课程中使用这些材料。

许多这些课程将会有之前使用过状态编程（在 Python、Java、Scratch 或其他语言中）的新生。根据我们的经验，这些学生中大多数人都得到了关于状态（例如，“变量是一个盒子”）的极其不完整或完全误导性的解释和隐喻。因此，他们对状态的理解仅限于绝对基础，特别是在他们到达别名等重要主题时。结果，许多这些学生发现，通过我们的概念机器正确理解状态是如何真正工作的，既新颖又深刻。因此，我们建议您缓慢而仔细地学习这些材料。

当然，我们邀请读者在章节内部创建他们自己的混合版本。我们很乐意听到其他人关于设计的想法。

### 1.8 我们选择编程语言的原因🔗 "链接到此处")

如果我们想致富，我们会用 Python 完全编写这本书。截至本书写作时，Python 正处于其教学用途的黄金时代（就像之前的 Java、C++、C、Pascal 等一样）。确实，Python 有许多吸引人的方面，其中最不重要的是它在职位列表旁边的存在。然而，我们反复对 Python 作为学习编程的入门点感到沮丧。🔗

因此，这本书特色是包含两种编程语言。它从一个名为[Pyret](https://www.pyret.org/)的语言开始，这是我们为了解决我们的需求和挫折而设计的。它被明确设计为适合本书的编程风格，因此两者可以和谐共生。它借鉴了 Python，但也借鉴了许多其他优秀的编程语言。因此，初学者可以放心，他们正在得到照顾，而那些对语言动物园（从蛇到骆驼）有所了解的程序员应该会发现 Pyret 既熟悉又舒适。

然后，认识到 Python 作为标准通信语言以及其广泛库的价值，本书的使用状态编程（在 Pyret 和 Python 中）部分明确涵盖了 Python。我们不是从 Python 的零开始，而是从早期材料中系统地、逐步地过渡到 Python。我们相信这将比只看到一种编程语言更好地让你学习通用编程。然而，我们也相信这将帮助你更好地理解 Python：就像你一旦走出国门，接触到其他语言、国家或文化，你会更好地欣赏自己的语言、国家或文化一样。

### 1.9 编程工具🔗 "链接到此处")

[CPO](https://code.pyret.org/)（code.pyret.org 的缩写）是我们为 Pyret 定制的默认编程环境。它完全在浏览器中运行，并使用 Google Drive 进行身份验证和文件存储。

Pyret 也支持 VSCode。要安装它，请访问 [`code.visualstudio.com`](https://code.visualstudio.com)。安装后，点击左侧的扩展选项卡，搜索 Pyret。您应该安装“Pyret Interactive Editor for VSCode”。

CPO 和 VSCode 在加载电子表格数据方面的支持不同。CPO 设计用于与 Google Sheets 一起工作，而 VSCode 不支持 Google Sheets，而是期望学生从本地 CSV 文件加载数据。除非书中另有说明，所有代码都可以在任一工具中运行，编辑和与程序交互的界面是相同的。如果有差异，书中将展示两种版本。在这种情况下，只能在 VSCode 中运行的代码将带有 VSCode 图标，如下所示：

> ```py
> include csv
> ```

我们不推荐任何特定的 Python 环境。任何允许您使用 pytest 和加载外部数据文件的 Python 编辑器都应该可以正常工作。
