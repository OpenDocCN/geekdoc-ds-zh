# 前言

在过去五年里，计算机领域发生了一场革命，背后推手是连续多年崭露头角的顶级游戏硬件制造商——NVIDIA。随着 CUDA（计算统一设备架构）编程语言的推出，这些强大的图形协处理器首次被日常的 C 程序员用于卸载计算密集型的工作。从嵌入式设备行业到家庭用户，再到超级计算机，这一切都因而发生了变化。

计算机软件行业的一个重大变化是从串行编程到并行编程的转变。CUDA 在这一变化中发挥了巨大作用。图形处理单元（GPU）天生就是为高速图形处理设计的，而图形处理本质上是并行的。CUDA 采用了简单的数据并行模型，并将其融入到编程模型中，无需图形原语。

实际上，CUDA 与其前辈不同，不需要了解图形或图形原语。您也不必是游戏程序员。CUDA 语言使 GPU 看起来就像另一个可编程设备。

本书假设读者对 CUDA 或并行编程没有任何预备知识。我假设他们只具备 C/C++编程语言的基础知识。随着我们深入，您将逐步掌握 CUDA，并且我们会讲解更多高级主题，将您从一个不了解并行编程的程序员，培养成能够充分发挥 CUDA 潜力的程序员。

对于已经熟悉并行编程概念和 CUDA 的程序员，我们将详细讨论 GPU 架构，以及如何充分利用每一代 GPU，包括最新的 Fermi 和 Kepler 硬件。实际上，任何能用 C 或 C++编程的人，只需稍加培训，几小时内就能掌握 CUDA 编程。从新手 CUDA 程序员，经过几倍的加速，到最终实现 10 倍以上的加速，这是本书结束时您应该能够做到的目标。

本书非常注重学习 CUDA，但重点在于性能优化，首先要确保程序的正确性。您对编写高性能代码，特别是 GPU 代码的技能和理解，将从本书中受益匪浅。

本书是一本关于在实际应用中使用 CUDA 的实用指南，由真正的实践者编写。然而，同时我们也会涉及必要的理论和背景知识，让每个人都能跟得上，无论背景如何，都能学会如何使用 CUDA 编程，这使得本书非常适合专业人士以及正在学习 GPU 或并行编程的读者。

本书内容安排如下：

**第一章****：超级计算的发展简史**。本章是对流处理器演变的广泛介绍，涵盖了一些关键的进展，帮助我们了解今天的 GPU 处理技术。

**第二章****: 使用 GPU 理解并行性。** 本章是并行编程概念的介绍，讲述了串行程序和并行程序的区别，以及如何通过不同的方式解决问题。本章主要面向已有串行编程经验的程序员，为后续章节中的概念提供理解基础。

**第三章****: CUDA 硬件概述。** 本章提供了对 CUDA 设备内部和周围硬件及架构的详细解释。为了从 CUDA 编程中获得最佳性能，必须对设备内外的硬件有一个合理的理解。

**第四章****: 设置 CUDA。** 本章讲解了在 Windows、Mac 和 Linux 变种下安装和设置 CUDA SDK 的过程。同时我们还将介绍 CUDA 的主要调试环境。

**第五章****: 网格、块和线程。** 本章详细解释了 CUDA 线程模型，并通过示例展示了在此模型中的选择如何影响性能。

**第六章****: CUDA 中的内存管理。** 理解不同的内存类型以及它们在 CUDA 中的使用方式是影响性能的最大因素。本章详细解释了各种内存类型的工作原理，并提供了示例，说明了错误使用这些内存类型的陷阱。

**第七章****: 实际应用中使用 CUDA。** 本章详细探讨了中央处理单元（CPU）和图形处理单元（GPU）如何最佳合作解决多种问题，以及 CPU/GPU 编程中的相关问题。

**第八章****: 多 CPU 和多 GPU 解决方案。** 本章探讨了如何在应用程序中编程和使用多个 GPU。

**第九章****: 优化你的应用程序。** 本章详细分解了 CUDA 中限制性能的主要因素，并探讨了可用的工具和技术，用于分析 CUDA 代码。

**第十章****: 库和 SDK。** 本章介绍了一些 CUDA SDK 示例和随 CUDA 提供的库，以及如何在应用程序中使用这些库。

**第十一章****: 设计基于 GPU 的系统。** 本章探讨了构建自己的 GPU 服务器或集群时涉及的一些问题。

**第十二章****: 常见问题、原因和解决方案。** 本章介绍了大多数程序员在开发 CUDA 应用程序时常犯的错误，以及如何发现和避免这些错误。
