1.  F 论文

建立对材料理解的一种方式是使用它。这些论文的目的是给你一个机会在现实世界的环境中实施你所学的知识。完成论文对于构建求职申请的简历也很重要。

预期每年都会变化，因此请将“以往示例”视为示例而不是模板。

## F.1 *唐纳德森* 论文

### F.1.1 任务

+   以**独立**和完全可重复的方式进行工作，请在[Open Data Toronto](https://open.toronto.ca)上找到一个感兴趣的数据库，并撰写一篇简短的论文讲述数据的故事。

    +   创建一个组织良好的文件夹，并添加适当的子文件夹，然后将其添加到 GitHub。你应该使用这个[起始文件夹](https://github.com/RohanAlexander/starter_folder)。

    +   在[Open Data Toronto](https://open.toronto.ca)上找到一个感兴趣的数据库。（虽然不被禁止，但请勿使用有关大流行的数据集。）

        +   汇编一个 R 脚本，“scripts/00-simulate_data.R”，模拟感兴趣的数据库并开发一些测试。推送到 GitHub，并包含一个有信息的提交信息。

        +   编写一个 R 脚本，“scripts/01-download_data.R”，以可重复的方式使用`opendatatoronto`（Gelfand 2022）下载实际数据。保存数据：“data/raw_data/unedited_data.csv”（使用有意义的名称和适当的文件类型）。推送到 GitHub，并包含一个有信息的提交信息。

    +   使用 Quarto“paper/paper.qmd”准备一个 PDF，包括以下部分：标题、作者、日期、摘要、引言、数据和参考文献。

        +   标题应该是描述性、信息性和具体的。

        +   日期应该采用明确的格式。在致谢中添加 GitHub 仓库的链接。

        +   摘要应该是三到四句话。摘要必须告诉读者主要发现。我们因为这篇论文而了解到的关于世界的一件事是什么？

        +   引言应该是两到三段内容。并且应该有一个额外的最后一段，概述论文的其余部分。

        +   数据部分应该彻底和精确地讨论数据的来源以及产生它的更广泛背景（伦理、统计等）。使用文本、图表和表格全面描述和总结数据。图表必须使用`ggplot2`（Wickham 2016）制作，表格必须使用`tinytable`（Arel-Bundock 2024）制作。图表必须显示实际数据，或尽可能接近实际数据，而不是总结统计。图表和表格应在文本中进行交叉引用，例如，“表 1 显示……”。

        +   应使用 BibTeX 添加参考文献。务必引用 R、你使用的任何 R 包以及数据集。优秀的论文将参考相关文献并引用它们。

        +   论文应写得很好，引用相关文献，并解释所有技术概念。针对受过教育但非专业人士的受众进行撰写。

        +   使用附录来包含支持性但非关键性的材料。

        +   推送到 GitHub 并包含一个信息性的提交信息

+   提交 GitHub 仓库的链接。请勿在截止日期后更新仓库。

+   不应有这是课堂作业的证据。

### F.1.2 检查

+   最终 PDF 中不应包含 R 代码或原始 R 输出。

+   README 中关于 LLM 使用的一个示例声明，你可以基于此编写自己的声明是：“LLM 使用声明：代码的部分是用自动完成工具 Codriver 编写的。摘要和引言是用 ChatHorse 编写的，整个聊天记录可在 `other/llm/usage.txt` 中找到。”

+   论文应直接渲染为 PDF，即使用“渲染为 PDF”。

+   图表、表格和文本应该清晰，并且与《金融时报》的质量相当。

+   日期应该是最新的且无歧义的（例如，2/3/2024 是模糊的，2 March 2024 则不是）。

+   整个工作流程应该是完全可重复的。

+   不应有任何拼写错误。

+   文中不应有任何迹象表明这是一篇学校论文。

+   必须使用脚注链接到论文的 GitHub 仓库。

+   GitHub 仓库应该组织良好，并包含一个信息性的 README。

+   论文应写得很好，能够被《金融时报》等报纸的普通读者理解。这意味着你可以使用数学符号，但必须用普通语言解释所有内容。所有统计概念和术语都必须解释清楚。你的读者是受过大学教育的人，但并不一定理解 p 值是什么。

+   摘要需要“紧凑”地撰写，几乎要简洁。删除不必要的词语。不要包含超过四句话。（一旦你有了经验，你可以打破这个规则。）

+   引言需要段落（在 Quarto 文档中行与行之间留空格）。

+   在引言中，请简要介绍论文的其他部分：“第二部分……，第三部分……”。（一旦你有了经验，你可以打破这个规则。）

+   请不要从他们的服务器读取数据到 Quarto 文档中，读取保存的版本。这样做提交的论文将得到 0 分。

+   在引言中，请更具体地说明你的发现。

+   数据部分不是关于数据清洗的，而是关于数据的。将数据清洗放入附录中。除非有重要内容，否则不要在数据部分讨论数据清洗。

+   模拟需要一个种子。

+   不要将仓库命名为“Paper 1”或类似的名称。

+   不要有“图表”或“表格”或类似的章节。

+   使用 `usethis::git_vaccinate()` 获取更好的 gitignore 文件，并且特别用于忽略 `DS_Store`。

+   请记住引用你使用的数据集以及 `opendatatoronto` - 它们是两回事。

### F.1.3 常见问题解答

+   我可以使用 Kaggle 的数据集吗？不可以，因为他们已经为你做了艰苦的工作。

+   我不能使用代码下载我的数据集，我能否手动下载？不可以，因为你的整个工作流程需要可重复。请修复下载问题或选择不同的数据集。

+   我应该写多少内容？大多数学生提交的内容在两到六页之间，但由你决定。要精确和全面。

+   我的数据是关于公寓楼/NBA/英雄联盟的，所以没有更广泛的环境，我该怎么办？请重新阅读相关的章节和阅读材料，以更好地理解偏见和伦理。如果你真的想不出什么，那么选择不同的数据集可能值得考虑。

+   我可以使用 Python 吗？不可以。如果你已经知道 Python，那么学习另一种语言并不会有什么坏处。

+   为什么我需要引用 R，而不需要引用 Word？R 是一个具有学术起源的免费统计编程语言，因此承认他人的工作是很合适的。这也对于可重复性很重要。

+   我应该使用哪种引用风格？任何主要的引用风格都可以（APA、Harvard、Chicago 等）；只需选择你习惯的一种。

+   起始文件夹中的论文有一个模型部分，所以我需要组装一个模型吗？不需要。起始文件夹被设计成适用于所有论文；只需删除你不需要的部分。

+   起始文件夹中的论文有一个数据表附录，所以我需要准备一个数据表吗？不需要。起始文件夹被设计成适用于所有论文；只需删除你不需要的部分。

+   “绘制实际数据”是什么意思？如果你在数据集中有，比如说 5,000 个观测值和三个变量，那么对于每一个变量，都应该有一个图表，在点的情况下有 5,000 个点，在条形图和直方图的情况下总和为 5,000。

## F.2 *Mawson* 论文

### F.2.1 任务

+   作为一到三人的团队的一部分，请选择一个你感兴趣的文章，其中代码和数据可以从以下来源获得：

    1.  自 2019 年以来发表的任何论文，发表在美国经济学会[期刊](https://www.aeaweb.org/journals)。这些期刊包括：“美国经济评论”、“AER：洞察”、“AEJ：应用经济学”、“AEJ：经济政策”、“AEJ：宏观经济学”、“AEJ：微观经济学”、“经济文献杂志”、“经济视角杂志”、“AEA 论文与会议论文集”。

    1.  在[此处](https://i4replication.org/reports.html)可用的任何来自复制研究所的文章，其可重复性状态为“寻找复制者”。

    1.  [吉尔·费尔德曼](https://mgto.org/check-me-replicate-me/)的一篇论文。¹

+   根据[*社会科学计算可重复性加速指南*](https://bitss.github.io/ACRE/)，请至少从该论文中复制至少三个图表、表格或其组合，使用[社会科学复制平台](https://www.socialsciencereproduction.org)进行复制。注意你的复制件的 DOI。

+   以完全可重复的方式进行工作，然后基于论文的两个或三个方面进行复制，并撰写一篇关于此的简短论文。

    +   创建一个组织良好的文件夹，包含适当的子文件夹，将其添加到 GitHub，然后使用 Quarto 准备一个包含以下部分的 PDF（你应该使用这个[起始文件夹](https://github.com/RohanAlexander/starter_folder)）：标题、作者、日期、摘要、引言、数据、结果、讨论和参考文献。

    +   你在论文中关注的方面可能与复制的方面相同，但不必如此。遵循论文的方向，但使其成为你自己的。这意味着你应该提出一个稍微不同的问题，或者以稍微不同的方式回答相同的问题，但仍然使用相同的数据库。

    +   在你的论文中包含你复制件的 DOI 以及支撑你论文的 GitHub 仓库的链接。

    +   结果部分应传达发现。

    +   讨论应包括三个或四个子部分，每个部分都关注一个有趣的观点，还应有一个子部分讨论论文的弱点，另一个子部分讨论其潜在的下一步。

    +   在讨论部分以及任何其他相关部分，请确保讨论伦理和偏见，并参考相关文献。

    +   论文应该写得很好，参考相关文献，并解释所有技术概念。针对受过教育但非专业人士的受众。

    +   使用附录来支持，但不是关键材料。

    +   代码应该是完全可复制的，有良好的文档，并且易于阅读。

+   提交 GitHub 仓库的链接。请勿在截止日期后更新仓库。

+   应该没有证据表明这是一项课程作业。

### F.2.2 检查

+   论文不应只是从原始论文中复制粘贴代码，而应将其作为工作的基础。

+   您的论文应该链接到相关的 GitHub 仓库和您进行的社会科学复制平台复制的 DOI。

+   确保您已经引用了所有内容，包括在讨论（以及其他部分）中引用的相关文献。参考文献的风格不重要，只要保持一致即可。

### F.2.3 常见问题解答

+   我应该写多少内容？大多数学生提交的内容在 10 到 15 页之间，但由您决定。要精确且全面。

+   我是否必须专注于模型结果？不，此时最好避开这一点，而是专注于总结或解释性统计的表格或图表。

+   如果我选择的论文不是用 R 语言写的怎么办？您的复制和复制代码都应该用 R 编写。因此，您需要将代码翻译成 R 进行复制。复制应该是您自己的工作，因此也应该用 R 编写。一种常见的语言是 Stata，Huntington-Klein (2022)可能作为 R、Python 和 Stata 的某种“罗塞塔石碑”很有用，或者直接使用 LLM 来帮助。

+   我可以独立工作吗？是的。

+   图表/表格是否必须与原文完全相同？不，您欢迎，并且应该使它们在复制过程中看起来更好。即使在复制过程中，它们也不必完全相同，只要足够相似即可。

+   我的一个图表有四个面板，我是否必须全部完成才能将其计为一个元素？不，对于这篇论文的目的，每个面板都算作一个单独的元素，所以您只需要完成三个面板就足够了。

+   如果数据在登录后才能访问，我如何自动下载？如果数据在登录后才能访问，只需在`download_data.R` R 文件中添加注释文档说明如何下载，而不是代码。

+   如果原始数据真的很大，我们是否需要将其提交到 GitHub？不，如果数据太大，你不必一定将原始未编辑的数据提交到 GitHub，只需在 README 中添加说明情况以及如何获取数据的说明即可。

+   摘要和引言应该关于什么？摘要和引言应该反映你自己的工作和发现，而不是原始论文的（尽管这些发现必然会有所作用）。你（几乎肯定）没有复制他们的整个论文，因此你的摘要应该是不同的。请参考示例以获取指导。

## F.3 *Howrah* 论文

### F.3.1 任务

+   作为一至三人团队的一部分，并以完全可重复的方式进行，请从[美国综合社会调查](https://gss.norc.org/Get-The-Data)³获取数据。（您可以使用不同的政府运行调查，但在开始之前请获得许可。）

+   获取数据，专注于调查的一个方面，然后用它来讲述一个故事。

    +   创建一个组织良好的文件夹，包含适当的子文件夹，将其添加到 GitHub，然后使用 Quarto 准备包含这些部分的 PDF（您应该使用这个[起始文件夹](https://github.com/RohanAlexander/starter_folder)）：标题、作者、日期、摘要、引言、数据、结果、讨论、至少包含一个调查的附录，以及参考文献。

    +   除了传达感兴趣的数据集的感觉外，数据部分应包括但不限于：

        +   讨论调查的方法论及其关键特征、优点和缺点。例如：调查对象、抽样框和样本；如何招募样本；采用何种抽样方法，以及这种方法的某些权衡；如何处理非响应。

        +   讨论问卷：它的优点和缺点是什么？

        +   如果内容过于详细，则使用附录来支持但不必要的内容。

    +   在附录中，请准备一个补充调查，可以用来增强论文关注的综合社会调查。补充调查的目的是获取关于论文关注主题的额外信息，这些信息超出了综合社会调查所收集的信息。该调查将以与综合社会调查相同的方式分发，但需要独立进行。补充调查应使用调查平台准备。应在附录中包含该链接。此外，还应包括调查的副本。

    +   请务必讨论伦理和偏见，并参考相关文献。

    +   代码应完全可重复、有良好的文档和可读性。

+   提交 GitHub 仓库的链接。请在截止日期后不要更新仓库。

+   论文应写得很好，参考相关文献，并解释所有技术概念。面向受过大学教育但非专业人士的读者。使用调查、抽样和统计术语，但务必解释清楚。论文应流畅，易于跟随和理解。

+   应没有证据表明这是一篇课程论文。

### F.3.3 常见问题解答

## F.7 *Spofforth* 论文

### F.7.1 任务

+   作为一至三人团队的一部分，请使用后分层多级回归预测即将到来的美国总统选举的普选票，然后写一篇简短的论文讲述这个故事。

+   这需要个人层面的调查数据、后分层数据以及将它们结合在一起的模式。鉴于收集这些数据的成本以及访问它们的特权，请确保正确引用您使用的所有数据集。

+   您将需要：

    +   获取个人层面的调查数据集的访问权限。

    +   获取后分层数据集的访问权限。

    +   清理和准备这两个数据集，以便它们可以一起使用。

    +   使用调查数据集估计模型。

    +   将训练好的模型应用于后分层数据集以预测选举结果。

+   您应使用此[起始文件夹](https://github.com/RohanAlexander/starter_folder)。

+   提交 GitHub 仓库的链接。请勿在截止日期后更新仓库。

+   应该没有证据表明这是一篇课堂论文。

### F.8.3 常见问题解答

+   我可以作为一个团队工作吗？不可以。您必须有一些完全属于自己的工作。您真的需要自己的工作来展示给求职申请等。
