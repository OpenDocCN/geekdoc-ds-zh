# 数论

> 原文：[`en.algorithmica.org/hpc/number-theory/`](https://en.algorithmica.org/hpc/number-theory/)

在 1940 年，一位英国数学家 G. H. 哈代（G. H. Hardy）发表了一篇著名的论文，题为“[数学家的辩护](https://en.wikipedia.org/wiki/A_Mathematician%27s_Apology)”讨论了数学应该为了自身的目的而不是为了其应用而追求的观点。

与数学类似，计算机科学的各个领域也形成了一个光谱，一端是数学逻辑和可计算性理论，另一端是网络编程和应用开发。我假设你，读者，更倾向于应用方面：这本书的写作是为了表明，在实用算法设计方面工作的人实在太少，而不是在理论计算机科学方面——而且既然你已经读到第七章，你很可能也相信这个说法。

然而，无论个人对这个问题的看法如何，人们都可以看到哈代（Hardy）的出发点。在写作时，他已经 62 岁，亲眼目睹了第一次和正在进行的第二次世界大战造成的破坏——这种破坏在很大程度上是由于科学的武器化。作为一名数论学家，哈代发现在一个“无用”的领域工作并无需面对任何道德困境，他写道：

> 到目前为止，还没有人发现数论或相对论有任何战争目的，而且似乎在许多年内也不太可能有人这样做。

具有讽刺意味的是，这个说法在原子弹发展后的 5 年内就被证明是错误的，而原子弹的制造离不开对相对论的理解[了解相对论](https://en.wikipedia.org/wiki/Einstein%E2%80%93Szil%C3%A1rd_letter)，以及计算机时代的密码学的诞生，这些密码学在很大程度上建立在数论之上——而数论的计算方面正是本章的主要内容。
