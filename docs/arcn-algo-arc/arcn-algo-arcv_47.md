# 密码学

> 原文：[`www.algorithm-archive.org/contents/cryptography/cryptography.html`](https://www.algorithm-archive.org/contents/cryptography/cryptography.html)

人类几乎总是对发送只有发送者和接收者才能理解的秘密信息感兴趣。原因很明显：秘密信息应该保持秘密。这种情况发生的最简单方式是在闭门谈话，但如果发送者和接收者相隔很远，那就行不通了。在这种情况下，他们需要依赖信使或邮差来发送信息。

为了简化，让我们假设他们是为了在古希腊或古罗马时期谈判战争策略而发送书面信件。显然，如果发送者和接收者都信任信使，信息可以保持秘密；然而，如果信使实际上是一个邪恶的间谍怎么办？如果信使被杀，信件被盗怎么办？如果（在精心策划的骗局中），某个第三方在深夜溜进信使的帐篷，用另一封完全不同的信件替换它怎么办？

这些都是密码学要解决的问题。

策略很简单：我们按照程序打乱我们发送的信息，并且只向受信任的各方提供解密程序。这样，除了发送者和接收者之外，其他人看到的信息就像是一堆乱码。即使信使是邪恶的，他们也无法阅读信息。如果信息被替换，也没有关系，因为接收者将无法正确解码信息，他们可以要求发送另一条信息（可能是在另一条路径上，由不同的信使发送）。不出所料，一种非常早期的加密方法据说是由凯撒（Julius Caesar）开发的，称为“凯撒密码”[[1]](#cite-1)。在这里，信息中的每个字符都根据发送者和接收者事先讨论的某个预定义的表格或图表替换为另一个字符。这个表格是通过简单地旋转字母表来创建的，其中  是在发送者和接收者之间的讨论中选择的。

这当然不是最复杂的方案，但通常这是人们在尝试相互编码秘密信息时首先想到的加密方案。说实话，我记得我在小学时和朋友互相发送信息，但我们从不会提供解码信息的必要表格。相反，我们会提供足够的文本，让他们自己从上下文中找到表格。如果一群小学生都能想出如何破解这个加密方案，那么它就不能太健壮。事实上，从凯撒密码被开发以来，密码学领域的发展非常有趣。在密码学文献中，总是有一个发送者、接收者和窃听者。出于我无法理解的原因，前两个人几乎总是被命名为爱丽丝（发送者）和鲍勃（接收者）。同时，攻击者或窃听者通常被称为伊芙或查理。这些名字甚至与量子密码学保持一致，所以它们将一直存在。

通常，有两种不同的加密类型：对称加密和非对称加密。这两种加密类型将在以下章节中描述。

密码系统是现代信息技术的基石，是 WiFi 到在线银行等一切事物的核心。如果攻击者设法破解现代密码算法，他们可能会造成严重损害。因此，重要的是要记住以下几点：

+   由于密码学已经成为一个如此先进的领域，因此密码系统应由受过培训的专业人士进行分析，并经过广泛的测试和审查。这意味着，只要可能，就应该使用广泛接受的密码学库，而不是编写自己的密码。

+   凯克霍夫原理指出，在确定密码系统的健壮性时，应假设攻击者知道加密和解密算法 [[2]](#cite-2)。这不包括任何预共享或秘密密钥。

+   随着技术的进步，密码学往往达到其极限。许多曾经是最先进的哈希算法已经过时，因为用于破解它们的计算机变得更快、更好。密码学将不得不面对的另一个领域是量子计算。量子计算机将对密码学产生重大影响，尤其是非对称密码学。这一系列问题在量子密码学领域得到了总结 [[3]](#cite-3)。

## 对称密码学

对称密码学被称为对称密码学，因为用于加密和解密的密钥是相同的。为了实现这一点，爱丽丝和鲍勃都需要相同的密钥，他们必须在通信之前共享这个密钥。对称密码学的例子包括：

+   **凯撒密码**：爱丽丝和鲍勃将字母表旋转了 3 个字符，并使用这个表格来编码和解码他们的信息 [[1]](#cite-1)。

+   **Rot13**：这是凯撒密码的一个特殊情况，其中字母表旋转了 13 位，因此得名“Rot13” [[4]](#cite-4)。

+   **置换密码**：在这里，我们选择一个置换（即）并根据该置换重新排列字母，这就是密钥 [[5]](#cite-5)。

+   **XOR 加密**：这种方法作用于位串，并用 XOR 运算符将消息和等长位的密钥结合起来 [[6]](#cite-6)。要解密，只需用相同的密钥再次进行 XOR 运算。

+   **DES 或数据加密标准**：这是一个较新的加密算法，于 1977 年标准化 [[7]](#cite-7)。自那以后，它被认为是不安全的，并被 AES 取代。

+   **AES 或高级加密标准**：实际的算法被称为“Rijndael” [[8]](#cite-8)。与 XOR 或 DES 类似，我们生成一个位串（取决于你使用哪种 AES，长度为 128/192 或 256 位）作为你的密钥。

+   **Blowfish**：这个算法也曾是 AES 的有力竞争者，但最终输给了 Rijndael [[9]](#cite-9)。

## 非对称加密

非对称加密有时被称为“公钥加密”（或简称为 PK 加密）因为鲍勃和爱丽丝都需要一个共享的公钥和私钥，他们自己保留。这些算法被称为非对称的，因为用公钥加密的内容只能用私钥解密，反之亦然。这可以用于多种不同的应用，如数字签名、加密通信或秘密共享密钥。例如，如果爱丽丝想给鲍勃发一条消息，并且这条消息必须保密，爱丽丝会用鲍勃的公钥加密这条消息。现在只有鲍勃才能再次解密这条消息并阅读它。如果查理想要篡改爱丽丝的消息，鲍勃就再也不能解密了。如果鲍勃想要确保这条消息确实是来自爱丽丝的，爱丽丝可以用她的私钥再次加密已经加密的消息。这是为了防止查理发送伪造或篡改的消息，因为鲍勃无法用爱丽丝的公钥解密那一层。公钥加密的一些例子：

+   **RSA**：这个算法从两个非常大的质数中计算出公钥和私钥 [[10]](#cite-10)。在可行的时间内，几乎不可能分解这两个质数的乘积。

+   **ECC 或椭圆曲线加密**：在这里，你从椭圆曲线上的两个点计算出私钥和公钥 [[11]](#cite-11)。这个正面的副作用是，与 RSA 等非 ECC 算法相比，你需要的数字更小，就能达到相同的安全级别。
